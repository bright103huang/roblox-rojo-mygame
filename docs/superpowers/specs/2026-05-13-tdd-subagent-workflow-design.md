# 子代理 TDD 强制执行方案设计

**日期:** 2026-05-13
**状态:** 待实施

## 问题

子代理 CLAUDE.md 中虽然写了"严格 TDD 流程"，但：
1. 没有指定**如何运行测试**（子代理不知道 `tests/run.lua` 的存在）
2. 没有要求**提取纯逻辑**（Roblox 依赖代码无法直接测试）
3. 主 agent 没有**强制审核**测试证据

## 方案

### 核心模式：提取纯逻辑层测试

```
┌─────────────────────────────────────┐
│  纯函数（纯 Luau，可测试）              │  ← 写测试 + luau tests/run.lua 验证
│  - 计算公式、数据校验、状态转换           │
│  - 入参出参都是基本类型/table             │
└────────────┬────────────────────────┘
             │ 被调用
┌────────────▼────────────────────────┐
│  Roblox 胶水层（薄，不可测试）           │  ← 只做参数提取 + 调用纯函数 + Roblox API
│  - 读 Player Attributes              │
│  - FireClient/FireServer             │
│  - 创建 Instance                     │
└─────────────────────────────────────┘
```

### 改动 1：CLAUDE.md 子代理 Prompt 模板

替换原模板（第 158-171 行）为：

```text
你是子代理，负责 [认知域名称]。
任务：[任务名称]

## TDD 强制流程（必须遵守）
1. 分析任务，识别可提取的纯逻辑（计算公式/数据校验/状态转换）
2. 在 tests/run.lua 底部添加 describe/it 测试用例（RED case）
3. 运行验证 RED：cd 项目根目录 && luau tests/run.lua → 预期新测试 FAIL
4. 实现纯函数（GREEN）
5. 再次运行 luau tests/run.lua → 预期 ALL PASS
6. 编写 Roblox 胶水层调用纯函数（保持最薄）
7. 将测试结果（stdout 输出）返回主 agent

## 提取纯逻辑原则
- 纯函数 = 入参出参都是基本类型/table，不依赖任何 Roblox API
- Roblox 胶水层只做：读参数 → 调纯函数 → 写回结果/FireEvent
- 如果任务没有可提取的纯逻辑（纯 UI 绑定等），需说明原因

## 约束
- 禁止修改 tests/run.lua 已有测试（只追加新的 describe）
- 禁止修改其他子代理的文件
- 遇到问题必须上报，禁止跳过步骤
```

### 改动 2：CLAUDE.md 主 agent 强制行为加一条

在 `# 🔒 主 agent 强制行为` 区域追加：

> * **TEST EVIDENCE**：子代理必须提供 `luau tests/run.lua` 测试通过的证据，否则禁止合并

### 改动 3：tests/run.lua 头部加 TDD 模板注释

在文件头部添加注释，让子代理一看就明白如何写测试。

## 不采用方案

### 不采用 lune
Lune 是独立 Luau 运行时，但不提供 Roblox API（Instance、Players、RemoteEvent 等），引入它并不能解决 Roblox 依赖代码的测试问题。我们的方案是通过架构分离（提取纯逻辑）来测试，而非引入新运行时。

### 不采用单独的测试文件
当前 `tests/run.lua` 单个文件足够容纳所有纯逻辑测试。等超过 500 行再拆分。保持简单。

## 影响范围

- **只修改配置文档**，不修改任何游戏代码
- 对子代理的工作流增加"写测试→跑测试"步骤，但每个任务仅增加约 5-10 分钟
- 长期收益：减少回归 Bug，每个模块都有测试覆盖
