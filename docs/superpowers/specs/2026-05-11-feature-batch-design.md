## 1. 传菜订单生成时机

### 改动
将订单生成从 `Drop` 成功时改成 `Pick` 成功时。

### 流程
```
旧: Pick(无订单) → 取盘 → Drop → 订单生成 → 标记目标桌
新: Pick → 生成订单 → 显示目标桌 → 送餐 → Drop → 结算
```

### 影响
- `DeliverTask.OnPlayerPickup()` 内调用 `assignDeliveryTarget()`，成功后返回 `true`，盘子+桃子在 Pick 阶段生成
- `DeliverTask.OnPlayerDrop()` 去掉订单生成逻辑，只做结算
- `TaskService` 无需修改

### 玩家体验变化
- 取餐时立即看到目标桌子（圈圈标记），送餐路径清晰
- 不会出现"拿到菜不知道往哪送"的茫然

## 2. 妖兽战场重做：三击碰撞系统

### 实体可视化
- 妖兽用 Part 代码组装（保留原有 Wolf/Tiger/Snake/Dragon 四种造型）
- **Bug 修复:** `buildAnimalShape()` 中 Parts 未设置 CFrame，需为每个 Part 计算 `root.CFrame * CFrame.new(offset)`，使部件正确分布在空间上（原代码中所有 Part 重叠在原点）
- Boss 出场时体型放大（`Scale` 从 1→1.3，Dragon 已配置），增加视觉压迫感
- 为保持简洁，每个 tier 固定一种动物形态：
  - Normal → Wolf / Tiger / Snake 随机
  - Boss → Dragon（唯一）

### 战斗机制：冲锋碰撞
```
Round 1: 妖兽自动从右侧铁门处冲锋到玩家面前 → 碰撞闪白 → 弹回
Round 2: 玩家需要走近妖兽（18格以内）→ 妖兽冲锋 → 碰撞闪白 → 弹回
Round 3: 同上（走近 → 冲锋 → 碰撞）
结算: 80% 胜利（得奖励），20% 遇到更强妖兽（受伤）
```

### 原有代码保留
- 光环系统（旋转 Neon 球体）：保留但视觉上调弱（透明度和大小可调整）
- 冲撞闪光：保留，碰撞时闪白恢复
- `CollisionAnimation()`：冲锋→碰撞→弹回三段动画保留

### 所需修改
- `buildAnimalShape()` 修复 CFrame 偏移
- `RingFlash` 闪光效果保留
- `FightSequence` 保持 3 轮结构

## 3. 妖兽负面效果：妖气侵蚀

### 机制
每次击杀妖兽，玩家累积 **Risk（妖气）**。Risk 影响后续妖兽战的难度：

| Risk 区间 | 状态 | 精英概率 | Boss 概率 | 暴走概率 |
|-----------|------|---------|----------|---------|
| 0-30 | 风平浪静 | 0% | 0% | 0% |
| 30-60 | 妖气隐现 | 5% | 0% | 0% |
| 60-80 | 危机四伏 | 15% | 5% | 0% |
| 80-100 | 大凶之兆 | 30% | 15% | 5% |

### 数值配置
```lua
Risk = {
    BaseGain = 8,              -- 普通击杀
    EliteGain = 12,            -- 精英击杀  
    BossGain = 20,             -- Boss 击杀
    DecayPerInterval = 1,      -- 每 120s 自然衰减
    DecayInterval = 120,       -- 秒
    MinRisk = 10,              -- 最低降到 10
}
```

### 暴走逃逸
暴走妖兽（5%）不奖励，会追玩家到其他场景。触摸即伤。玩家需逃离或等待其 15 秒后消失。

### 与已有系统的关系
- Risk 值存储在 DataManager 中（已有 `Risk` 字段）
- 服务端 `BeastTask.OnAttack()` 中 roll 妖兽等级时检查 `Risk` 值
- 衰减纳入 `StatusService` 的定时循环

## 4. 时间系统昼夜压力

### 四时辰设计

| 时辰 | 游戏时段 | 工作消耗 | 炼丹 | 冥想 | 战斗 |
|------|---------|---------|------|------|------|
| 晨 | hour 4-12 | -15% | +15% | - | 遇精英概率+10% |
| 昼 | hour 12-18 | 正常 | 正常 | 正常 | 正常 |
| 暮 | hour 18-22 | +20% | - | - | +20% 伤害 |
| 夜 | hour 22-28(4) | +40% | - | 恢复×1.4 | +40% 伤害 |

### 提示
- 每个时辰变更时，客户端显示简短提示语（古风风格，2 秒自动消失）
- 示例："卯时·旭日东升 —— 炼丹效率提升 15%"
- "亥时·夜深人静 —— 打坐恢复提升 40%"

### 午夜结算（凌晨 0:00，hour=0）
- Fatigue -30
- TotalDays +1
- 若 Malice > 50，额外扣除 GongDe（戾气反噬）
- 触发环境光照切换（晨光/正午/黄昏/深夜）

### 消耗倍率实现
- `StatsConfig.TIME_MODIFIERS` 已定义了 4 个时间段的 `taskEff`（消耗倍率）
- `StatusService.ApplyCosts()` 中根据当前时辰应用倍率
- 注意：任务成本表中的数值是基础值，乘以倍率后扣除

### 所需修改
- 确认 `TIME_MODIFIERS` 数值与设计一致
- 客户端增加时辰切换提示（复用 AlchemyUI 弹窗风格）
- `ApplyCosts` 中加入倍率逻辑

## 5. 玩家可见反馈统一

### 现状问题
- 炼丹：精美面板（AlchemyUI），有动画、图标、颜色
- 传菜：只有 `print()`，玩家看不到任何信息
- 妖兽：只有 `print()`，无视觉反馈
- 时辰切换：当前无任何提示

### 解决方案
抽取轻量提示工具 `TaskNotice.lua`（ModuleScript），被 `TaskClient.local.luau` require：

```lua
TaskNotice:Notify({
    Title = "标题",
    Text = "内容",
    Color = Color3.fromRGB(g, r, b),  -- 主题色
    Duration = 2,  -- 秒
})
```

渲染方式：简单半透明 Frame + TextLabel，居中偏上，自动淡出。

### 触发场景
| 事件 | 标题 | 文本示例 | 颜色 |
|------|------|---------|------|
| 传菜接单 | 传菜 | "送餐到 XXX" | 金色 |
| 传菜完成 | 传菜 | "仙晶 +10，功德累计 +1/5" | 金色 |
| 妖兽战斗 | 角斗场 | "第 1 轮碰撞！" | 红色 |
| 妖兽胜利 | 角斗场 | "击败 XX！仙力经验 +8，仙晶 +20" | 红色 |
| 妖兽失败 | 角斗场 | "妖兽太强！体力 -10" | 深红 |
| 时辰变更 | 卯时 | "旭日东升 · 炼丹效率 +15%" | 根据时辰 |
| 午夜结算 | 子时 | "天地更替 · 疲劳 -30" | 蓝色 |

## 6. 仙丹阁

### 场景与 NPC
- 位置：现有 `DanShop` 场景坐标 `(-1000, 3, 0)`
- 布局：柜台（长桌 Part，Z=0）+ NPC 售货员造型（Part 组装，类似妖兽但更小）
- 触摸 NPC 或柜台 → 弹出 `ShopUI.local.lua`

### 商品清单

| 丹药 | 售价 | 效果 | 解锁条件 | 定位 |
|------|------|------|----------|------|
| 回气丹 | 20 仙晶 | 体力 +20 | 无 | 日常补给 |
| 清心散 | 35 仙晶 | 疲劳 -15 | 无 | 过劳应急 |
| 聚神丹 | 40 仙晶 | 精神 +15 | 无 | 炼丹前补充 |
| 玉灵丹 | 80 仙晶 | 火毒 -20 | 无 | 中毒救命 |
| ??? 修为小丹 | 200 仙晶 | 身法/仙力/火候 随机+1 | 仙晶≥200 解锁信息 | 修为突破（打工/氪金） |
| ??? 修为大丹 | 500 仙晶 | 三项全+1，无视经验 | 仙晶≥500 解锁信息 | 硬核突破 |

### 朦胧机制
- 后两种修为丹默认显示为 `???`，图标灰色，描述隐藏
- 当玩家当前仙晶 ≥ 售价时，该丹药自动"解封"显示名称和效果
- 造成"原来还有这种好东西！"的惊喜感
- 即使玩家花掉仙晶跌回售价以下，已解锁的丹药保持可见（只检查一次）

### 购买流程
1. 触摸 NPC/柜台 → `TaskEvent:FireClient("OpenShop", { items = ... })`
2. `ShopUI.local.lua` 显示商品列表
3. 玩家点击购买 → `TaskEvent:FireServer("Buy:DanShop", { itemId = "..." })`
4. 服务端 `ShopService` 检查仙晶是否足够 → 扣款 → 应用效果
5. 回执购买结果

### 价格参照
- 传菜一次 10 仙晶 → 2 次传菜买 1 回气丹
- 精英妖兽奖励 50 仙晶 → 1 次精英怪买 3 回气丹
- 修为小丹 200 仙晶 → 20 次传菜 / 4 次精英怪 → 需要主动存钱
- 修为大丹 500 仙晶 → 50 次传菜 → 长期目标
